## CIX-Adaptive-AR-Interface – Unity Project Setup & Usage

This repository contains the **CIX-Adaptive-AR-Interface** Unity project. Follow the steps below to set it up locally, open it in Unity, and run / build it.

### 1. Prerequisites

- **Unity Hub** installed on your machine  
- **Unity Editor** version compatible with this project  
  - Open `ProjectSettings/ProjectVersion.txt` to see the exact Unity version, or let Unity Hub detect it automatically.
- **Git** installed and configured with your GitHub account (for contributing / syncing changes)
- (If you use speech features) Internet access and credentials for any external speech service (Azure, Google Cloud, etc.)

### 2. Required Unity Packages & External Dependencies

Open the project once in Unity to let it restore everything automatically. Then verify:

- **Unity Packages** (via **Window → Package Manager**):
  - Any **Text‑to‑Speech (TTS)** or **Speech‑to‑Text (STT)** packages used by the project (e.g. plugins in `Assets/Plugins` or packages listed in `Packages/manifest.json`) are installed without errors.
  - All other packages referenced by the project are resolved (no red error icons in Package Manager).
- **External SDKs / Services** (only if this project uses them):
  - Install the SDKs or Unity plugins required by your speech provider (e.g. Azure Cognitive Services, Google Cloud Speech, etc.).
  - Configure service keys and endpoints in a way that is **not committed to git** (e.g. via scriptable objects in an ignored folder, environment variables, or local config files).

If the project relies on specific speech packages or SDKs, document them here (name + version + link) so that anyone can reproduce the setup.

### 2.1 API Keys & Secrets (IMPORTANT)

- **Never** hard‑code API keys (OpenAI, Wit.ai, Azure, etc.) directly in:
  - C# scripts (e.g. `AdaptiveOrchestrator.cs`)
  - Unity scenes (e.g. `SampleScene.unity`)
  - Any asset or config file that is tracked by git.
- Instead, always:
  - Store keys in **environment variables** or in **local config assets/files** that are ignored by git.
  - Add any local secret/config files to `.gitignore` so they never appear in commits.
- If you ever accidentally commit a key:
  - **Immediately rotate/regenerate** the key in the provider’s dashboard.
  - Remove it from the code and from git history before pushing again (GitHub push protection will help you catch leaks).

### 3. Cloning the Project

1. **Create a folder** where you want to keep your Unity projects.
2. **Clone the repository**:

```bash
git clone https://github.com/SinaElahimanesh/CIX-Adaptive-AR-Interface.git
cd "CIX-Adaptive-AR-Interface"
```

Now you have a local copy of the **CIX-Adaptive-AR-Interface** project under your GitHub username.

### 4. Opening the Project in Unity

1. Open **Unity Hub**.
2. Click **"Add"** (or **"Open"** depending on the version).
3. Select the cloned project folder (the one that contains `Assets`, `Packages`, `ProjectSettings`, etc.).
4. Unity Hub will detect / install the required Editor version if needed.
5. Open the project from Unity Hub.

### 5. Running the Project (Play Mode)

Once the project is open in Unity:

1. Open the main scene (commonly under `Assets/Scenes/`).
2. In the Unity Editor, press the **Play** button at the top.
3. Use the **Game** view to interact with the running project.

### 6. Building the Project

To produce a standalone build or mobile build:

1. In the Unity Editor, go to **File → Build Settings…**
2. Select the target platform (e.g., **PC, Mac & Linux Standalone**, **Android**, **iOS**, etc.).
3. Ensure the correct scene(s) are added to **Scenes In Build**.
4. Click **Build** (or **Build And Run**).
5. Choose an output folder (commonly a `Build` or `Builds` folder which is ignored by git).

### 7. Project Structure (High Level)

- **`Assets/`**: All game / app content: scripts, scenes, prefabs, shaders, etc.
- **`Packages/`**: Package references (managed by Unity’s Package Manager).
- **`ProjectSettings/`**: Unity project-wide settings (input, graphics, etc.).
- **`UserSettings/`**: Per-user editor settings (ignored by git).
- Generated folders like **`Library/`**, **`Logs/`**, **`Temp/`**, and build output folders are ignored via `.gitignore` to keep the repository size reasonable.

### 8. Contributing / Working with Git

Typical workflow:

```bash
git pull          # get latest changes
# make your changes in Unity / scripts
git status        # see what changed
git add .         # stage changes
git commit -m "Describe your change"
git push          # push to GitHub
```

Avoid committing:

- `Library/`, `Temp/`, `Logs/`, `UserSettings/`  
- Build outputs (e.g., `.apk`, `.aab`, `Build/`, `Builds/`)  
- Crash dumps and other large generated files

These are excluded by `.gitignore` and can always be regenerated by Unity.

### 9. Troubleshooting

- **Project takes very long to open**: This can happen the first time because Unity is rebuilding the `Library/` folder. Subsequent loads will be faster.
- **Missing packages / errors after opening**:  
  - Go to **Window → Package Manager** and let Unity resolve / download all dependencies.
  - Make sure you are using the Unity version specified in `ProjectVersion.txt`.
- **Unexpected files showing up in git**:  
  - Run `git status` and verify that no `Library/`, `Temp/`, build artifacts, or user-specific settings are being tracked.
  - If needed, update `.gitignore` accordingly.
